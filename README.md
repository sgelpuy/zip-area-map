# 우편번호 구역 지도

우편번호(5자리, 국가기초구역 기준)를 여러 개 입력하면 카카오맵 위에 해당 구역 경계를 색깔별로 그려줍니다.

- 데이터 출처: 우정사업본부 "구역의도형" (`TL_KODIS_BAS`, 기초구역=현행 우편번호 경계), EPSG:5179 → WGS84 변환
- 전국 17개 시도, 34,516개 우편번호 구역 포함
- 순수 정적 사이트 (백엔드 서버 없음): `data/<앞2자리>/<우편번호>.json` 형태로 구역별 좌표를 잘라서 저장해두고, 입력한 우편번호만 그때그때 fetch

## 로컬에서 보기

카카오맵 JavaScript SDK는 등록된 도메인에서만 동작합니다 (localhost/127.0.0.1은 등록 불가). 로컬에서 지도 렌더링까지 확인하려면 실제 배포 도메인(GitHub Pages 등)을 카카오 디벨로퍼스에 등록해야 합니다.

```
python -m http.server 8080
```

## 데이터 갱신 / 지역 추가

1. `../구역의도형_전체분_<지역명>/` 폴더에 우정사업본부 구역의도형 shapefile을 압축 해제해서 넣기
2. `python convert.py` 실행 → `data/` 폴더가 갱신됨
3. 커밋 후 푸시하면 GitHub Pages에 자동 반영

## 우편번호별 예상 세대수

행정안전부 주민등록 인구·세대 현황(2026년 6월 기준, 행정동 단위)을 우편번호 구역에
**면적 비례 배분(areal interpolation)** 방식으로 나눠서 각 우편번호의 예상 세대수를 추정했다
(`population_join.py`). 방법:

1. 행정동 경계(`TL_SCCO_GEMD`, 10자리 행정동코드)와 우편번호 경계(`TL_KODIS_BAS`)를 원본 좌표계(EPSG:5179, 미터)에서 교차
2. 우편번호 구역이 특정 행정동과 겹치는 면적 비율만큼, 그 행정동의 세대수를 나눠서 배분
3. 여러 행정동에 걸쳐 있으면 각 행정동에서 배분받은 값을 합산

**한계 / 정확도**: 이 방법은 행정동 안에서 세대가 균등하게 분포한다고 가정한다. 실제로는
아파트단지·녹지·상업지구 등으로 밀도가 균일하지 않기 때문에:
- 우편번호 구역이 행정동 하나를 통째로 포함하면 → 정확히 그 행정동 통계와 일치 (오차 0)
- 우편번호 구역이 여러 행정동에 걸쳐 잘게 겹칠수록 → 오차 커짐 (해당 지역 세대 분포가 불균일할수록 더 커짐)

우편번호 단위의 실제 세대수 통계가 공식적으로 존재하지 않아 정답과 비교해 "정확도 몇 %"를
계산할 방법이 없다. 대신 배분 방식 자체의 정합성은 확인했다: 전국 합계(약 2,419만 세대)가
행정동 통계 합계와 정확히 일치하고(총량 보존), 세종시로 예비 검증했을 때도 우편번호별 합산이
원본 행정동 합계(164,401세대)와 소수점 오차 이내로 일치한다. 즉 "행정동 내부 밀도가 균일하다"는
가정이 맞는 지역일수록 이 추정치는 정확하고, 그 가정이 깨지는 지역(같은 행정동 안에 아파트단지와
공터/산이 함께 있는 경우 등)에서는 실제보다 과대/과소 추정될 수 있다.
